## Brief overview
Content and communication preferences for the Wheat Bunny bakery website project. These guidelines ensure consistent messaging, tone, and branding across the site.

## Communication style
- Keep text concise and to the point
- Avoid verbose descriptions; shorter is better
- Use active, warm, and personal tone
- Emphasize passion and hobby over commercial business language

## Content messaging priorities
- Highlight "home baker" rather than "business" or "company"
- Emphasize simplicity: flour, salt, water as core ingredients
- Mention highest quality ingredients without being overly detailed
- Include versatility: suitable for any meal, snack, or gift
- Avoid unverified claims (e.g., don't claim "eco-friendly" unless certain)

## Product descriptions
- Keep descriptions brief but evocative
- Use sensory words (buttery, chewy, crispy) where appropriate
- Include ingredients list in overlay, not main description
- Show prices clearly in SGD with weights/sizes

## Design presentation
- Ensure images display fully on mobile without cropping
- Use responsive CSS that shows complete product images
- Avoid fixed heights that cut off image content on smaller screens
- Allow natural image aspect ratios to display properly

## Development workflow
- Do NOT modify files in `/release/` directory - this is handled by `make release`
- Make changes to root level files only (index.html, styles.css)
- Use category organization for menu items (Sourdough Breads, Desserts, etc.)
- Test responsive behavior on mobile viewport sizes
